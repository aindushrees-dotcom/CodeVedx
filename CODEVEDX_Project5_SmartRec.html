How It Works — Content-Based Filtering
The engine encodes both user preferences and movie metadata into 24-dimensional feature vectors (14 genre dimensions + 8 mood dimensions + rating + era), then computes cosine similarity between the user vector and each item vector. The angle between vectors determines how closely preferences align with movie attributes.
01
Feature Encoding
User preferences (genres, moods, rating, era) and movie metadata are both converted into normalized binary vectors in the same 24-dimensional feature space.
02
Cosine Similarity
cos(θ) = (A · B) / (‖A‖ × ‖B‖) Measures the angle between two vectors. A score of 1.0 means perfect alignment; 0 means no overlap.
03
Weighted Scoring
Final Score = 0.45 × Genre + 0.35 × Mood + 0.12 × Rating + 0.08 × Era Weights reflect the relative predictive power of each feature.
04
Ranked Output
All 20 movies are scored and sorted descending. The top 8 are returned as personalized recommendations with full per-factor breakdowns.
